# H5支付系统集成文档

## 概述

本项目已集成新的H5支付系统，支持支付宝H5、微信H5和信用卡支付。新系统提供了更完善的支付流程和状态管理。

## 系统架构

### 1. 配置管理

**文件**: `src/utils/appConfig.js`

```javascript
// 是否启用新的H5支付系统
export const USE_H5_PAYMENT = true;

// H5支付系统基础URL  
export const H5_PAYMENT_BASE_URL = 'http://localhost:8080/payment/h5';
```

### 2. API接口封装

**文件**: `src/api/h5Payment.js`

主要接口：
- `createH5Payment(data)` - 创建H5支付订单
- `queryPaymentStatus(orderSn, paymentChannel)` - 查询支付状态
- `cancelPayment(orderSn, paymentChannel)` - 取消支付
- `getPaymentMethods(type)` - 获取支付方式列表

### 3. 支付页面

**文件**: `src/pages/money/h5Pay.vue`

新的H5支付页面特性：
- 动态获取可用支付方式
- 支持信用卡信息输入和验证
- 支付状态轮询
- 支付结果处理

### 4. 支付管理工具

**文件**: `src/utils/paymentManager.js`

提供统一的支付管理功能：
- 支付页面跳转管理
- 支付渠道映射
- 支付状态格式化

## 支付流程

### 完整支付流程

1. **订单创建** → **选择支付方式** → **支付处理** → **状态查询** → **支付完成**

### 详细步骤

#### 1. 创建订单
- 用户在 `createOrder.vue` 页面提交订单
- 系统调用 `/order/generateOrder` 接口创建订单
- 根据 `USE_H5_PAYMENT` 配置决定跳转到哪个支付页面

#### 2. 支付方式选择
- 页面加载时调用 `getPaymentMethods()` 获取可用支付方式
- 用户选择支付渠道（支付宝H5/微信H5/信用卡）
- 如果选择信用卡，显示信用卡信息输入表单

#### 3. 创建支付订单
- 调用 `createH5Payment()` 接口创建H5支付订单
- 传递订单信息、支付渠道、信用卡信息等
- 后端返回支付URL或支付结果

#### 4. 支付处理
- **支付宝/微信H5**: 跳转到第三方支付页面
- **信用卡**: 直接处理支付结果
- 对于H5支付，启动状态轮询机制

#### 5. 状态查询
- 每2秒调用 `queryPaymentStatus()` 查询支付状态
- 根据状态（PENDING/SUCCESS/FAILED/CANCELLED）采取相应行动
- 支付成功后跳转到成功页面

#### 6. 支付完成
- 更新本地存储标记，通知其他页面刷新
- 跳转到 `paySuccess.vue` 页面显示结果

## 支付渠道说明

### 支持的支付渠道

| 渠道代码 | 渠道名称 | 描述 |
|---------|---------|------|
| ALIPAY_H5 | 支付宝H5支付 | 适用于移动端浏览器的支付宝支付 |
| WECHAT_H5 | 微信H5支付 | 适用于移动端浏览器的微信支付 |
| CREDIT_CARD | 信用卡支付 | 预下单信用卡支付 |

### 支付状态

| 状态代码 | 状态名称 | 描述 |
|---------|---------|------|
| PENDING | 待支付 | 支付订单已创建，等待用户支付 |
| SUCCESS | 支付成功 | 支付已完成 |
| FAILED | 支付失败 | 支付过程中出现错误 |
| CANCELLED | 已取消 | 用户取消支付 |

## 信用卡支付特殊处理

### 信用卡信息字段

```javascript
creditCardInfo: {
    cardHolderName: '',  // 持卡人姓名
    cardNumber: '',      // 卡号（自动格式化）
    expiryMonth: '',     // 过期月份（MM）
    expiryYear: '',      // 过期年份（YYYY）
    cvv: '',            // CVV安全码
    billingAddress: ''   // 账单地址
}
```

### 验证规则

- 持卡人姓名：必填
- 卡号：16位数字，自动添加空格格式化
- 过期月份：01-12
- 过期年份：不能小于当前年份
- CVV：3-4位数字
- 账单地址：必填

## 兼容性设计

### 支付系统切换

通过 `USE_H5_PAYMENT` 配置可以在新旧支付系统之间切换：

```javascript
// 在订单相关页面中
if (USE_H5_PAYMENT) {
    // 使用新的H5支付系统
    uni.navigateTo({
        url: `/pages/money/h5Pay?orderId=${orderId}`
    });
} else {
    // 使用原有支付系统
    uni.navigateTo({
        url: `/pages/money/pay?orderId=${orderId}`
    });
}
```

## 错误处理

### 常见错误场景

1. **网络错误**: 显示通用错误提示，允许重试
2. **支付方式加载失败**: 显示错误提示，提供刷新选项
3. **信用卡验证失败**: 显示具体验证错误信息
4. **支付创建失败**: 显示后端返回的错误信息
5. **支付状态查询失败**: 继续轮询，避免用户感知

### 错误处理策略

```javascript
// 统一错误处理函数示例
handlePaymentError(error) {
    let errorMessage = '支付失败，请重试';
    
    if (error.data && error.data.message) {
        const message = error.data.message;
        if (message.includes('库存不足')) {
            errorMessage = '商品库存不足，无法完成支付。请返回购物车重新选择商品。';
        } else if (message.includes('信用卡')) {
            errorMessage = '信用卡信息验证失败，请检查后重试';
        } else if (message.includes('订单')) {
            errorMessage = '订单异常，请联系客服处理';
        } else {
            errorMessage = message;
        }
    }
    
    uni.showModal({
        title: '支付失败',
        content: errorMessage,
        showCancel: true,
        confirmText: '重试',
        cancelText: '返回',
        success: (res) => {
            if (!res.confirm) {
                uni.navigateBack();
            }
        }
    });
}
```
